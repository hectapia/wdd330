Getting Started with JSON Web Tokens (JWT)


The article is titled Get Started with JSON Web Tokens1. It is a tutorial that explains what JSON Web Tokens (JWTs) are, how they are structured, and how they can be used for authentication and information exchange.
The article defines JWTs as compact and self-contained JSON objects that can be securely transmitted between parties. The article also states that JWTs can be verified and trusted because they are digitally signed using a secret or a public/private key pair.
The article then describes the structure of a JWT, which consists of three parts separated by dots: header, payload, and signature. The article explains that the header contains the type of the token and the hashing algorithm, the payload contains the claims (statements about an entity or metadata), and the signature is calculated using the header, the payload, and the secret or key.
The article also shows how to decode a base64url encoded JWT to read its information, and how to verify its signature using an online tool or a library. The article provides a diagram and some examples of JWTs and their decoded values.



The article then discusses some scenarios where JWTs are useful, such as:
Authentication: Once the user is logged in, each subsequent request will include the JWT, allowing the user to access routes, services, and resources that are permitted with that token.
Single Sign On: JWTs can be used to implement SSO across different domains and systems, because they are small and can be easily used among them.
Information Exchange: JWTs can be used to securely transmit information between parties, because they can be signed and verified.
The article concludes by providing some links and resources for further learning about JWTs, such as a free ebook, a video course, and a playground.

Develop, Debug, Learn?


The video is titled Develop, Debug, Learn? A time to re-think our tooling. It is performed by Christian Heilmann, a developer advocate at Microsoft. The video argues that we need to change our approach to web development and debugging to be more empathetic and user-centric.
The video starts by pointing out that we have a lot of tools and frameworks to help us code faster and more efficiently, but we often neglect the quality and usability of our final products. The video claims that we are too focused on our own preferences and workflows, and we forget about the needs and expectations of our end users and other developers.  
The video then suggests some ways to improve our web development and debugging practices, such as:
Using progressive enhancement to build for the lowest common denominator and then enhance for more capable browsers and devices.
Testing our products on different browsers, devices, networks, and environments to ensure they work for everyone.
Using browser developer tools to inspect and debug our code, as well as learn from other websites and web standards.
Using VS Code as an integrated development environment that allows us to debug web projects end-to-end with a dedicated browser instance and access to browser developer tools inside the editor.
Using Edge DevTools for VS Code extension to get more features and functionality from the Edge Developer Tools inside VS Code.
Using launch.json files to automate the process of starting the browser and opening the localhost address for debugging.
The video concludes by encouraging us to re-think our tooling and workflows to be more empathetic and user-centric, and to share our knowledge and feedback with others.



What about the problem of context switching?


What about the problem of context switching? As both Visual Studio Code and Edge are based on Chromium, we can do something about this. Using the the Elements for VS Code extension, you can embed the developer tools of the browser directly in the editor. The editor fires up an instance of the browser and hides it in the background. That way you can use both the editor and the visual tools in the same context.
