Object Methods, This


The web page explains how to use object methods in JavaScript, which 
are functions that are stored as properties of an object. Object methods can access the 
object data using the this keyword, which refers to the current object. 
The web page also shows how to use method shorthand syntax, which is a 
shorter way to write object methods. The web page provides some examples of object methods 
and how they work with different types of property names and values. 
The web page is part of a tutorial on modern JavaScript.


  
    let calculator = {
      sum() {
        return this.a + this.b;
      },
    
      mul() {
        return this.a * this.b;
      },
    
      read() {
        this.a = +prompt('a?', 0);
        this.b = +prompt('b?', 0);
      }
    };
    
    calculator.read();
    alert( calculator.sum() );
    alert( calculator.mul() );
  
  
  


  
This in JavaScript

The web page explains how the this keyword works in JavaScript, which is a common source of confusion for many developers. The web page shows that the value of this depends on how a function is called, and there are six different contexts where this can take on different values. They are:

    

  1. this in global context: this refers to the Window object in the browser.
    2. 

  2. this in object construction: this refers to the new instance of the object created with the new keyword.
    3. 

  3. this in an object method: this refers to the object itself, which allows accessing its properties and methods.
    4. 

  4. this in a simple function: this refers to the global object (Window) by default, but can be changed with methods like call, apply, and bind.
    5. 

  5. this in an arrow function: this refers to the same value as the enclosing scope, which is determined by where the arrow function is defined, not where it is called.
    6. 

  6. this in an event listener: this refers to the element that received the event, unless the listener is an arrow function or bound to a different object.
    7. 


The web page provides examples to illustrate how this works in each context, and also gives some tips and best practices on how to use thiscorrectly and avoid common pitfalls.
